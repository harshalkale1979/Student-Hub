How Student Hub works

A plain-language guide to what this site does, where your data lives, and how to put it online with GitHub Pages — written for a first-time GitHub user.

Your data stays on your device

Every tool — GPA, attendance, planner, notes, timetable — saves what you type into your browser's local storage. Nothing is sent to a server, because this site doesn't have one. That also means: if you clear your browser data, or open the site in a different browser or device, your saved entries won't follow you. For anything important, it's worth keeping a backup copy elsewhere.

Put it on GitHub Pages — step by step

1

Create a GitHub account

Go to github.com and sign up if you haven't already. It's free.

2

Create a new repository

Click the + icon top-right → New repository. Name it something like student-hub. Keep it Public. Don't add a README — you already have one in this zip.

3

Upload your files

Unzip the file you downloaded. On your new repo's page, click uploading an existing file, then drag in everything — index.html, the assets folder, and the pages folder — keeping the same structure. Commit the changes.

4

Turn on GitHub Pages

Go to your repo's Settings tab → Pages (left sidebar) → under "Build and deployment", set Source to Deploy from a branch, branch main, folder / (root). Click Save.

5

Visit your live site

Wait about a minute, then refresh that same Pages settings tab — it'll show a link like https://yourusername.github.io/student-hub/. That's your live website, free forever, with no separate hosting needed.

Prefer the command line?

If you have Git installed, you can push the unzipped folder with:

cd student-hub
git init
git add .
git commit -m "Initial commit — Student Hub"
git branch -M main
git remote add origin https://github.com/YOUR-USERNAME/student-hub.git
git push -u origin main

Then turn on Pages the same way as Step 4 above.

Frequently asked

Can I customize the colors or text?

Yes. Open assets/css/theme.css and edit the values at the top under :root — that single file controls colors across every page. Page text lives directly inside each .html file.

Why don't my notes show up on my phone if I saved them on my laptop?

Because everything is stored locally in that one browser, on that one device. To sync across devices you'd eventually need a real backend database — outside the scope of this static starter, but a great next project once you're comfortable with the basics.

Can I add more subjects to the video library?

Yes — open pages/resources.html and find the subjects array near the bottom of the file. Add a new object with a name, color, and list of topics, following the same pattern as the existing ones.

The grading scale in the GPA calculator doesn't match my institute.

Open pages/gpa-calculator.html and find the gradePoints object near the bottom script. Change the letter grades and point values to match your institute's official scale.
